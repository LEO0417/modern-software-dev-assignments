# 教学行动指南（最高版本）

本指南用于统一教学理念、内容结构与落地流程，适用于两条主线的全部课程建设与复盘。

## 1. 教学定位
- 受众：零基础学习者，对 IDE、终端无概念，中文为主。
- 目标：通过项目导向学习，理解核心概念与逻辑，能读懂并复现项目。
- 核心能力：读懂代码、运行代码、改写代码、复盘过程。

## 2. 两条主线
- 主线 1（Learning Python with AI IDE）：为零基础提供完整知识铺垫，内容更细、更冗余，覆盖 Python + 终端 + AI IDE + Ollama + NG 相关基础。
- 主线 2（斯坦福 CS146S）：沿 Stanford Week1-Week8 节奏讲解理念与代码。
- 关系：主线 2 的知识点在主线 1 中有对应补齐；主线 1 的编排顺序对齐主线 2，并在关键节点推荐主线 2。
- 维护：两条主线必须同步在 `docs/MAPPING.md` 中维护映射表。

## 3. 叙述风格与结构
- 风格：讲义 + 练习，避免口播稿式叙事。
- 表达：短句、分层、概念先行；每个概念必须配最小示例。
- 连贯性：统一术语与命名（如环境名、工具名、模型名）。

## 4. 单课模板（建议）
- 课程目标（学完能做什么）
- 关键概念（术语、心智模型、常见误区）
- 演示与拆解（最小可运行示例 + 逐行解释）
- 练习（从仿写到改写到迁移）
- 复盘清单（学员自检）

## 5. 练习设计原则
- 由浅入深：先复现，再改写，最后迁移到小任务。
- 可验证：每个练习明确“运行后看到什么输出”。
- AI 协作：鼓励使用 AI 解释、定位报错、生成变体，但必须写出自己的改动。

## 6. 素材与引用
- Stanford 课程资料与 `week1` 现有脚本为主线 2 的主材。
- `ng_lesson` 作为并行参考，优先补齐主线 1 的 Python 基础。
- 引用时标注文件路径，避免无出处的信息拼贴。

## 7. 文件与命名规范
- 课程主线：`Learning Python with AI IDE/`（主线 1）。
- 斯坦福主线：`week1`-`week8`（主线 2，保持原结构）。
- 文档中心：`docs/`，包含行动指南、映射表、任务清单、工作日志。
- 旧内容：`week0` 视为历史素材，不再作为新主线模板。

## 8. 完成标准（Definition of Done）
- 课程具备可复现步骤（命令、文件、预期结果）。
- 术语一致，跨文档不冲突。
- 练习可独立完成并有答案检验方式。
- 映射表已更新，主线互引完整。

## 9. 更新流程
- 每日更新 `docs/WORKLOG/`。
- 每次新增或调整课程内容，必须同步更新映射表与任务清单。
- 每周复盘一次结构与进度，修正偏差。
