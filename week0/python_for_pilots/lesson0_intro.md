# Lesson 0: Taking the Wheel (掌舵 - 飞行员导论) ✈️ (口播稿)

> **场景建议**：背景音带有飞机起飞的轰鸣声，然后转为平稳的巡航噪音。
> **预计时长**：15 分钟
> **角色**：机长 (Captain/Instructor)

---

## 0. 开场 (Opening) - 3 min

**领航员**：
欢迎各位飞行员 (Pilots)。

恭喜你们从 Apprentice 阶段毕业。
你们已经向 **Stanford Modern Software Development Course** 迈出了坚实的第一步。
在那儿，你们学会了怎么把代码写得漂亮、规范。但这还不够。
仅仅在 Anti-gravity 的小黑框里打印 `Hello World`，改变不了世界。

今天，我们要进入 **Phase 2: Python for Pilots**。
在这个阶段，我们的关键词是 **"Automation" (自动化)** 和 **"Connectivity" (连接)**。

这也是斯坦福课程中非常强调的能力——我们要让 Python 走出那个封闭的盒子，通过 Anti-gravity IDE 去**控制**你的电脑，连接全世界的 AI 模型。

---

## 1. 我们的武库 (The Arsenal) - 5 min

**领航员**：
要成为一名合格的飞行员，你需要掌握三种新武器，它们将在未来的 Week 1 课程中扮演关键角色：

1.  **Modules & Pip (模块与包管理)**：
    斯坦福的课程是建立在巨人的肩膀上的。我们会大量使用现成的 AI 库。
    Anti-gravity 拥有最先进的依赖管理系统。我们要学的，就是怎么用 `pip` 命令，把这些神兵利器装到我们自己的飞船上。

2.  **APIs (应用程序接口)**：
    现代软件开发的核心就是 API。
    我们要学习怎么让你的 Python 代码给“气象局”发个信，或者给 **OpenAI** 发个信。
    在斯坦福课程中，你们将通过 API 构建强大的 RAG 系统。

3.  **Error Handling (异常处理)**：
    飞机在天上飞，难免遇到气流（Bug）。
    作为飞行员，你不能遇到问题就跳伞（崩溃）。我们要学 `try/except`，学会“单发着陆”。这是职业工程师的基本素养。

---

## 2. 核心任务：自动化 (The Mission) - 5 min

**领航员**：
在这个阶段，Anti-gravity IDE 将成为你的**副驾驶**。
你将不再问它“这个语法是什么意思”，而是开始给它下达战术指令：

> "Anti-gravity, write a script to monitor this website and alert me if the price drops."
> (写个脚本监控这个网站，降价了通过 API 通知我。)

这才是 Python 真正的威力。
我们将不再手动处理任何枯燥的任务。

---

## 3. 预告 (Teaser) - 2 min

**领航员**：
各位飞行员，检查你们的驾驶舱仪表盘。
下一节课 (Lesson 1)，我们将通过安装我们的第一个“外挂”——**External Libraries (外部库)** 来开始这段旅程。

你会发现，原来你可以只写 2 行代码，就做到以前写 100 行代码都做不到的事情。
这就叫 **Power (力量)**。

**Stanford Prep Phase 2 engaged. Pilots, cleared for takeoff.**
